EasyShop Master DevOps Pipeline

Global DevOps Lifecycle: Code -> Github Actions CI -> ArgoCD GitOps -> Kubeadm Cluster & Monitoring

1. CI Pipeline & Security

  • • Developer pushes code to GitHub triggering **GitHub Actions**.
  • • **Trivy Scanner** analyzes source files and Docker layers for vulnerabilities.
  • • Code is built into a unified image and pushed to **Docker Hub**.
  • • Security gates prevent insecure builds from pushing to registry.

2. GitOps & ArgoCD CD

  • • **ArgoCD Controller** runs in the cluster control plane namespace.
  • • Polls the GitHub repository config folder for manifest changes.
  • • Compares target state (git) with current state (cluster APIs).
  • • Automatically applies (syncs) changes into the active cluster namespaces.

3. Cluster Infrastructure & Monitoring

  • • Nodes created using **Terraform** EC2 structures and joined via **Kubeadm**.
  • • App runs as Next.js deployment accessing internal **MongoDB** database.
  • • **Promtail** sends logs to **Loki** which writes to AWS S3.
  • • **Prometheus** scrapes metrics, stores in EBS GP3, and routes to Grafana dashboards.
