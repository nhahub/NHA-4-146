EasyShop Ansible Provisioning & Monitoring Architecture

Visualizing playbook orchestration steps, control plane endpoint exposure, and the integrated log/metrics pipeline

Ansible Automation Flow

  • • Part 1 installs common prerequisites, kubectl, and setup packages across all hosts in the inventory.
  • • Part 2 installs Helm binaries and provisions GitOps ArgoCD components onto the Master Node.
  • • Part 3 builds chart dependencies and deploys the unified easyshop-monitoring stack (kube-prometheus-stack + loki-stack).

Kubeadm Secure Patches

  • • Exposes metrics by changing kube-scheduler and kube-controller-manager bind address to 0.0.0.0.
  • • Opens etcd metrics bind address to all interfaces on port 2381.
  • • Patches kube-proxy ConfigMap metricsBindAddress and rollouts the DaemonSet to apply changes.
  • • Appends --kubelet-insecure-tls to metrics-server deployment.

Integrated Observability

  • • Next.js frontend instrumented via prom-client exposing metrics at /metrics on port 3000.
  • • Node Exporter and Promtail DaemonSets scrape hardware and pod log files across all cluster worker nodes.
  • • Grafana configures Loki as a local datasource and dynamically imports pre-packaged developer dashboards.
