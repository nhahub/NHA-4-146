EasyShop Monitoring Pipeline

Self-Managed Kubernetes (Kubeadm) Observability Architecture on AWS EC2

1. Metric Scrape Pipeline

  • • Prometheus Server scrapes the host metrics and Kubernetes container statistics.
  • • Scrapes master/worker Kubelet APIs on port 10250.
  • • Uses Node Exporter DaemonSets on all cluster EC2 instances for server metrics.
  • • Persists processed metrics on AWS EBS GP3 volumes dynamically created by EBS CSI driver.

2. Loki Log Pipeline

  • • Promtail DaemonSet runs on all worker nodes and tails local system and container logs.
  • • Ships logs to Loki via HTTP POST JSON requests inside the cluster.
  • • Loki Aggregator indexes logs and pushes structured log chunks directly to AWS S3.
  • • Logs automatically expire in S3 after 30 days via S3 lifecycle rules.

3. Dashboard & Alerting

  • • Grafana UI connects to Prometheus and Loki sources for complete cluster visualization.
  • • Accessible via NodePort 30080 from the external Load Balancer.
  • • Prometheus routes system metrics warnings to Alertmanager.
  • • Alertmanager fires notifications to external destinations (Slack Webhooks/Email).
