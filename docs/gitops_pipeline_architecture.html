EasyShop GitOps & CI/CD

Detailed workflow: GitHub Actions CI (Trivy scanner), Docker Registry, and ArgoCD GitOps Git Reconciliation loop

1. CI Security Scan (Trivy)

  • • Push triggers a **GitHub Actions** Runner to test and build code.
  • • **Trivy Scanner** checks configuration files and package dependencies for vulnerabilities.
  • • Scans Docker base layers (`node:18-alpine`) before compiling the final production output.
  • • Validated builds build a standalone container and export to **Docker Hub** registry.

2. GitOps Git Poll (ArgoCD)

  • • **ArgoCD** runs locally inside the cluster control namespace (`argocd`).
  • • Listens or periodically polls the remote GitHub repository for changes.
  • • Uses git revision targets as the "Source of Truth" for cluster configurations.
  • • Initiates target reconciliations on the Kubernetes API to sync resources instantly.

3. Deployment Execution

  • • **Kube-API Server** receives the synced manifests (Deployments, services, configs).
  • • API scheduler assigns workloads and triggers Node **Kubelet** agents.
  • • Kubelet pulls the freshly pushed image from **Docker Hub** registry.
  • • Updates the containers under the `tws-e-commerce-app` namespace with zero downtime.
