EasyShop App & DB K8s Deployment

Kubernetes Runtime Topology: App Pods, MongoDB Database Services, and HostPath Volume mounts

1. Frontend App & NodePort Routing

  • • **e-commerce-service** routes external cluster visitor traffic.
  • • Exposes target Port `3000` externally on **NodePort 30080** on all node IPs.
  • • Pointing a client load-balancer to worker-ip:30080 directs traffic to Next.js containers.

2. Environmental Variables Injection

  • • **configmap.yaml** contains non-sensitive configs like `NEXT_PUBLIC_API_URL`.
  • • Fixed the API URL with `/api` suffix to avoid client axios route fallback.
  • • **secret.yaml** stores sensitive credentials (e.g. NextAuth Secret and Mongo Root credentials) in Base64 encoding.

3. Database Storage & Mounts

  • • MongoDB deploys inside the namespace, exposing port `27017` via a ClusterIP service named `mongodb`.
  • • Utilizes **mongodb-pvc** requesting `5Gi` storage to persist data.
  • • Resolves permissions: VM boots scripts pre-create `/mnt/data/easyshop-mongodb` and `chmod 777` to allow DB writing.
