EasyShop Kubeadm Infrastructure

Detailed AWS Virtual Network topology, Security Groups, and Master/Worker Kubeadm Bootstrap Flow

1. VPC Network Topology

  • • Network split into **Public Subnet** (hosting control plane) and **Private Subnet** (hosting application nodes).
  • • **NAT Gateway** allows worker nodes in the private subnet to fetch images/updates safely.
  • • **AWS S3** registers instance access flow logs for infrastructure auditing.

2. Ingress Security & Firewalls

  • • **sg-master** opens Port `6443` for kubectl/worker connection and ports `2379-2380` for internal etcd sync.
  • • **sg-worker** opens Port `10250` for metrics scraping and `30000-32767` for application ingress.
  • • CNI Calico handles overlay networks through encrypted **VXLAN UDP 4789** tunnels.

3. Automated Cluster Bootstrapping

  • • Master VM boot script runs `kubeadm init` and fetches join tokens.
  • • Terraform `local-exec` trigger calls `/bin/bash` with **join-cluster.sh** script.
  • • Script logs into worker VM via SSH and runs join bootstrap commands.
  • • Pre-creates `/mnt/data/easyshop-mongodb` with `chmod 777` to host database files safely.
